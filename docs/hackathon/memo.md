## ハッカソンの企画アイディアメモ

- 運営から与えられた架空のお題に対する最適な解を導き出すハッカソン
  - 架空の企業やお題を用意する
    - AWS Game day のイメージ
  - ハッカソンのコンセプト. 
    - AIによりハッカソンの形も変わってきた(ソロ参加とかも増えている)
    - 自ら指揮官となりAIを駆使して課題に対する最高のプロダクトを生み出せ！
    - ソロだと開発の他に設計やデザイン、プレゼンとか考えなきゃいけないけどAI時代だからこそそれができるはず！
    - 開発者の人もそうじゃない人もチャレンジとして取り組めるハッカソンになると良いかも！
  - ハッカソンの条件
    - 個人(ソロ)で参加すること
  - ハッカソンの流れ  
    - 1次フェーズと2次フェーズの2段階で分ける
    - 1次フェーズ(オンライン 2週間くらい？)
      - 与えられたお題に対する回答(プロダクト)を作成する
      - ここで一次審査を行う
      - 審査もオンライン(1週間くらいを想定)
      - スプシで点数など管理
      - 何チーム選出するかなどは全く未定(提出数にもよるが30チームくらいを想定)
    - 2次フェーズ(オフライン)
      - 1次フェーズ突破者のみ2次フェーズであるオフラインの1dayハッカソンに進める
      - 当日追加でお題を提供する
        - それを限られた時間でなんとか解決する
        - 当日は最終成果をピッチしてもらってその場で審査
      - ただし、デモ発表タイム時のみオープンで集客し、スポンサーセッションや懇親会の時間を設ける！
    - お題出しについて
