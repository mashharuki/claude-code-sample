## エグゼクティブサマリー

**一言で言うと**: 紙の経費精算を電子申請システムに切り替えることで、月末3日間の処理工数を90%削減し、年間約240万円相当のコストを回収する。

現在、当部署では毎月の経費精算を紙の申請書で運用しており、担当者が月末3日間（約24時間/月）を費やして処理しています。この運用には直接コストに加え、転記ミス・紛失リスク・承認遅延といった潜在的損失が発生し続けています。

本提案では、**クラウド型経費精算システムの導入**（既存IT資産を活用したIT部門との協働）により、申請から承認・支払いまでの全工程をデジタル化します。初期導入コストは約80万円、月次ランニングコストは約5万円と試算しており、導入後6ヶ月以内に投資を回収できる見込みです。

**承認いただきたいアクション**: 本提案の採択と、IT部門への要件定義開始の指示（目標: 2026年7月システム稼働）。

## 1. 背景・現状分析

### 1.1 現在の業務フロー

```
社員が紙の申請書を記入
      ↓
上司が紙で承認・押印
      ↓
担当者が手作業で集計・転記（Excel）
      ↓
経理部門へ紙+Excelで提出
      ↓
支払い処理（翌月）
```

### 1.2 業界トレンド

デジタル化への環境は整備されています。

- 2023年10月施行の**インボイス制度**により、適格請求書の電子保存・管理が事実上必須化
- 2024年1月施行の**電子帳簿保存法改正**により、電子データの紙出力保存義務が廃止され、電子保存が原則に
- 国内企業のクラウド型経費精算システム採用率は**2024年時点で約60%超**（IDC Japan調査）
- 同業他社・競合企業の多くがすでにデジタル化を完了

**法制度の観点からも、紙運用を継続するリスクが高まっています。**

## 2. 課題定義

### 2.1 本質的な問題

紙の経費精算運用は、「処理が遅い」という症状の背後に、3つの構造的問題を抱えています。

| # | 本質的な課題 | 現状の症状 |
|---|-------------|-----------|
| 1 | **工数の非効率** | 担当者が月末3日間（約24時間）を手作業集計・転記に費やす |
| 2 | **エラーリスクの常在** | 手書き・手転記による金額ミス、書類紛失が発生しうる |
| 3 | **コンプライアンスリスクの蓄積** | 電子帳簿保存法・インボイス制度への非対応で監査リスクが増大 |

### 2.2 定量的な損失試算

| 損失項目 | 試算 | 根拠 |
|---------|------|------|
| 担当者工数コスト | **月24時間 × 12ヶ月 = 年288時間** | 月末3日間×8時間 |
| 人件費換算（時給2,500円想定） | **年72万円** | 288時間 × 2,500円 |
| 上司承認の移動・押印工数 | 月4時間 × 12 = 年48時間 | 承認者5名×月均40分 |
| 申請者の記入・提出工数 | 月2時間 × 12 = 年24時間 | 社員20名×月均6分 |
| **工数コスト合計（概算）** | **年間約90万円超** | 上記合計 |
| エラー修正・差し戻し工数 | 月2〜3件発生と推定 | 実績ベース |
| 法令非準拠リスク（潜在損失） | 税務調査時のペナルティリスク | 電子帳簿保存法対応遅延 |

**「何もしない」コスト**: 現状を放置した場合、向こう3年間で270万円超の工数コストが累積します。

## 4. 企画コンセプト・提案内容

### 4.1 提案コンセプト

> **「月末3日間を取り戻す」— スマートフォン1つで完結する経費精算へ**

### 4.2 解決策の比較検討

3つのアプローチを比較し、**案B（クラウド型SaaS導入）を推奨**します。

| | 案A: Excelマクロ改善 | **案B: クラウド型SaaS（推奨）** | 案C: フルスクラッチ開発 |
|---|---|---|---|
| 初期コスト | 約10万円 | **約80万円** | 約500万円〜 |
| 月次コスト | ほぼ0 | **約5万円** | サーバー維持費等 |
| 導入期間 | 1ヶ月 | **2〜3ヶ月** | 6〜12ヶ月 |
| 工数削減効果 | 30〜40% | **90%** | 90%以上 |
| 法令対応 | 別途対応必要 | **標準対応済** | 要開発 |
| IT部門負荷 | 低 | **低〜中** | 高 |
| スマホ対応 | 不可 | **標準機能** | 要開発 |

**案Bを推奨する理由**: 初期コストは中程度ですが、工数削減効果・法令対応・保守性のバランスが最も優れています。SaaSのため自社IT部門の開発負荷を最小化でき、サービス提供会社が法令改正への対応を継続的に行います。

### 4.3 推奨システムの主要機能

- **スマートフォンでのレシート撮影・OCR自動読み取り**
- **ワークフロー（承認ルート）設定・自動通知**
- **会計システムとのデータ連携（CSV出力 / API）**
- **インボイス・電子帳簿保存法への標準対応**
- **管理画面でのリアルタイム集計・レポート**

## 5. 実施計画（アクションプラン）

### 5.1 フェーズ構成

本プロジェクトは3フェーズで推進します。

```
Phase 1: 要件定義・選定（2026年6〜7月）
  └── IT部門との要件定義、ベンダー選定、稟議決裁

Phase 2: 導入・設定・テスト（2026年8〜9月）
  └── システム設定、テスト運用、社員向けトレーニング

Phase 3: 本番稼働・定着（2026年10月〜）
  └── 全社運用開始、効果測定、継続改善
```

### 5.2 役割分担

| 役割 | 担当者 | 主な責務 |
|------|--------|---------|
| プロジェクトオーナー | 部門長 | 意思決定・予算承認 |
| PM（プロジェクトマネージャー） | 業務改善担当 | 進捗管理・調整 |
| IT部門担当 | IT部門 | 要件定義・ベンダー評価・システム連携 |
| 経理部門担当 | 経理部門 | 会計連携要件・承認フロー設計 |
| ユーザー代表 | 各部門代表社員 | テスト参加・フィードバック |

### 5.3 IT部門への依頼内容

IT部門には以下を依頼します。

1. **要件定義支援**: 既存会計システムとの連携要件の整理
2. **ベンダー評価**: セキュリティ要件（SOC2 / ISO27001等）の評価
3. **システム設定**: SSO（シングルサインオン）・ネットワーク設定
4. **データ移行支援**: 既存Excelデータの移行
5. **保守・運用体制**: 障害対応窓口の設定

## 10. 結論 — なぜ今やるべきか

### 実施する3つの理由

**1. コストの問題**  
現状維持は「安全」ではありません。毎月約7.5万円（年間90万円超）の工数コストが発生し続けています。3年間放置すれば270万円を超えます。今すぐ動くことが最も合理的な選択です。

**2. 法令対応の緊急性**  
電子帳簿保存法・インボイス制度への対応は今後ますます厳格化される見通しです。対応が遅れるほど、税務調査時のリスクが高まります。早期導入により、このリスクを先手で排除できます。

**3. 組織競争力の観点**  
同業他社の60%超がすでにデジタル化済みです。優秀な人材にとって、紙の経費精算は「この会社は古い」という印象につながります。デジタル化は採用・定着にも貢献します。

### 行動のお願い

**本日承認いただきたい事項**:

1. 本提案の採択（業務デジタル化の方針決定）
2. IT部門への要件定義開始の指示
3. 予算枠の確保（初期82.5万円 + 月次5万円）

**今すぐ動かない場合の損失**: 1ヶ月の先送りごとに約7.5万円の機会損失が生じます。2026年10月稼働を目指すには、**2026年6月中の意思決定**が必要です。
