# あなたの役割

あなたは、CSSやhtmxに精通したフロントエンドのエキスパートです。

以下の要件を満たすTodoアプリを開発してください。

# 要件
- Todoアプリは、ユーザーがタスクを追加、表示、完了、削除できる機能を持つこと。
- タスクの追加、完了、削除はページの再読み込みなしで行えること。
- タスクの状態（完了/未完了）は視覚的に区別できること（例：完了したタスクは取り消し線を引く）。
- タスクはローカルストレージに保存され、ページを再読み込みしてもデータが保持されること。
- なるべくシンプルかつモダンでかっこいいデザインにすること。
- htmxの中には分かりやすいコメントを入れること。
- `sample-htmx-todo-app`というフォルダを作成し、その中で作業を行うようにすること。

# 技術スタック
- htmx
- CSS（必要に応じてフレームワークを使用してもよい）

# API仕様
- モックAPIサーバーで良いです。

## バックエンドのエンドポイント
- `GET /api/todos`: すべてのタスクを取得する。
- `POST /api/todos`: 新しいタスクを追加する。リクエストボディにタスクの内容を含める。
- `PUT /api/todos/{id}`: 指定したIDのタスクの状態を更新する。リクエストボディに更新内容を含める。
- `DELETE /api/todos/{id}`: 指定したIDのタスクを削除する。
