# Intentional Design Guard

AI（LLM）はトレーニングデータの統計的中央値に収束する。このスキルは、その「分布収束」を意図的に破り、プロジェクトの文脈に合った唯一のデザインを生成するためのガードレールである。

---

## Blacklist: 絶対に使わないもの

以下は「AI が作った」と即座に見抜かれるシグナルである。代替なしに使用禁止。

### フォント
- Inter, Roboto, Open Sans, Lato, Montserrat, Poppins, Nunito, Source Sans Pro
- system-ui, -apple-system, Arial, Helvetica のみのフォントスタック

### カラー
- 白背景 + 薄い紫/青グラデーション (`#7C3AED` → `#3B82F6` 系)
- 黒背景 + ネオンパープル/シアン
- Tailwind デフォルトパレットの無加工使用 (`blue-500`, `purple-600` 等をそのまま)

### レイアウト
- 3カラム均等カードグリッドの反復
- Hero → Features (3列) → Testimonials → CTA の定型LP構成
- 意図なく `rounded-2xl shadow-lg` を全カードに適用

### コンポーネント
- アイコン + タイトル + テキストだけの Feature カード量産
- グラデーション背景の大型 CTA ボタン
- 装飾目的のみの半透明カード・ぼかし背景

---

## 必須プロセス: UI生成前の3問

コードを1行書く前に以下を自問し、答えをコード内コメントとして残す：

1. **誰が、何のために使う画面か？** → ユーザー像と目的を1行で
2. **この画面のムード/トーンは？** → 例: 信頼感、軽快さ、専門性、温かみ、緊張感
3. **類似しない参照点は？** → 同ジャンル外から1つインスピレーション源を選ぶ（例: 金融アプリ → 建築雑誌、教育ツール → レコードジャケット）

## タイポグラフィ

フォント選択は「文脈 → ムード → 書体」の順で決定する。

### 手順
1. 画面のトーンからムードキーワードを2つ選ぶ
2. `references/typography-alternatives.md` からムードに合うペアリングを選択
3. 見出しと本文でコントラストを確保する（ウェイト差 300+、またはセリフ×サンセリフの混合）

### 禁止
- 見出し・本文が同一ファミリー・同一ウェイト
- ウェイト差 200 以下（例: 400 vs 500）

---

## カラー

パレットは「コンセプト → 主色 → アクセント → ニュートラル」の順で構築する。

### 手順
1. コンセプトから出発点となる主色を1色決める（ブランドカラーがあればそれ）
2. 主色の色相を混ぜたニュートラル群を導出（純粋なグレーは使わない）
3. 補色関係または分割補色でアクセント色を1〜2色追加
4. CSS カスタムプロパティで定義し、Tailwind `extend` で登録する

### 禁止
- `#FFFFFF` 純白 + 彩度バラバラのカラフルカード群
- 意味なきグラデーション装飾
- パレット未定義のまま Tailwind ユーティリティで直接色ハードコード

### グラデーション代替テクニック
- 微細 SVG ノイズテクスチャ (`background-image: url("data:image/svg+xml,...")`)
- `backdrop-filter: blur()` + 半透明背景
- 幾何学 SVG パターン（ドット、ライン、クロスハッチ）
- `box-shadow` 多層レイヤーによる奥行き表現

---

## レイアウト

「意味のある不均一」を目指す。均等割りは思考停止のサイン。

### 原則
- グリッドに非対称を混ぜる（2:1, 3:2 比率、スパン変更）
- 余白を情報として扱う（余白が広い = 重要度が高い）
- セクション境界は余白のリズム変化で表現する（ラインやカード枠ではなく）
- コンポーネント配置は視線の流れに沿う（Z型、F型、または意図的な破壊）

### 禁止
- `grid-cols-3` 均等3列を2セクション以上で反復
- 全カード同サイズ・同構造
- `px-4 py-8` 等の余白パターンをページ全体で機械的に統一

---

## モーション

アニメーションは「意味のある動き」のみ許可する。

### 許可
- ユーザー操作フィードバック（ホバー、クリック、フォーカス）
- コンテンツ出現順を伝えるスタガーアニメーション
- 状態遷移を説明するトランジション（開閉、ページ遷移）

### 禁止
- 全要素がスクロールでふわっと出現する装飾フェードイン
- `animate-pulse`, `animate-bounce` の多用
- 意味なく `transition-all duration-300` を全要素に適用

## セルフレビューチェックリスト

UI実装後、以下をすべて確認する。1つでも該当すれば修正：

- [ ] Blacklist のフォントを使っていないか
- [ ] 紫グラデーション / ネオンパープルを使っていないか
- [ ] 3問（誰が、ムード、異分野参照）に答えているか
- [ ] 見出しと本文でフォントまたはウェイトに十分な差があるか
- [ ] カラーパレットが CSS 変数で定義されているか
- [ ] 均等3列カードを2箇所以上で繰り返していないか
- [ ] すべてのアニメーションに「なぜ動くのか」の理由があるか
- [ ] 画面全体を見て「AI が作った」と感じないか（最終主観チェック）
