# AWS Community Builder 継続審査（Renewal）ガイド

2年目以降の継続に向けた準備ガイド。

---

## 継続審査の概要

- **頻度**: 年1回の再申請が必要（自動更新ではない）
- **時期**: 通常10-12月頃（年によって変動あり）
- **審査基準**: Community Builder 期間中の貢献実績
- **結果通知**: 申請から1-3ヶ月後

## 審査で評価されるポイント

### 最重要: 残る形でのアウトプット

> 「残る形でアウトプットする」ことが継続に不可欠
> — classmethod.jp/aws-community-builder-2nd-year-maruto

1. **ブログ記事** - 最も評価されやすい。公開URLがエビデンスになる
2. **動画コンテンツ** - YouTube、勉強会の録画
3. **登壇スライド** - SpeakerDeck、SlideShare等で公開
4. **OSS コントリビューション** - GitHub上のPR/Issue

### 重要: コミュニティへの貢献

1. **Community Builders Slack での活動** - 質問回答、情報共有
2. **週次ウェビナーへの参加** - 出席自体が実績に
3. **ローカルコミュニティ活動** - JAWS-UG等への参加・運営
4. **他メンバーへのサポート** - メンタリング、レビュー

### 加点要素

1. **英語でのコンテンツ** - グローバルコミュニティへの貢献
2. **AWS認定資格の取得** - 技術力の証明
3. **大規模イベントでの登壇** - AWS Summit、re:Invent等
4. **コミュニティ運営** - JAWS-UG支部の立ち上げ・運営

## 申請書作成のコツ

### 自己紹介セクション
- AI Engineering カテゴリを選んだ理由を明確に
- 専門領域と情熱を端的に伝える
- 1年間の成長ストーリーを含める

### 活動実績セクション
- **数字で語る**: 「ブログ12本、登壇4回、イベント参加8回」
- **インパクトを示す**: 「記事の累計PV ○○、いいね ○○」
- **ベスト3リンクを厳選**: 最も自信のある活動を選ぶ

### 今後の計画セクション
- 具体的な活動計画を述べる
- コミュニティへの還元意欲を示す
- AWS Heroes への道のりを意識した目標設定

## タイムライン（逆算スケジュール）

| 時期 | アクション |
|------|-----------|
| **申請6ヶ月前** | 活動実績の棚卸し。不足分野を特定 |
| **申請4ヶ月前** | 不足分野の活動を集中的に実施 |
| **申請2ヶ月前** | ベスト3リンクの候補を絞り込み |
| **申請1ヶ月前** | 申請書の下書き作成 |
| **申請2週間前** | 他のCBメンバーにレビュー依頼 |
| **申請直前** | 最終チェック・提出 |

## 参考: 1年目から2年目で活動量が増えた例

classmethod の記事より:
- ブログ: 8本/年 → **32本/年**（4倍）
- 登壇: 4回/年 → **11回/年**（約3倍）
- コミュニティ: 参加のみ → **JAWS-UG支部を立ち上げ**

**ポイント**: 1年目の経験と人脈を活かして2年目に爆発的に活動量を増やせる。

## 万が一不合格の場合

- 翌年に再申請可能
- 不合格理由は通知されないことが多い
- 活動量を増やして再チャレンジする
- 他の CB メンバーにアドバイスを求める
