# 分析フレームワーク実践ガイド

業界課題を構造的に発見・整理するためのフレームワーク集。
各フレームワークの目的・使い方・組み合わせ方を解説する。

---

## フレームワーク適用の順序

```
外部環境(マクロ)           業界構造             自社/対象企業
    ↓                      ↓                    ↓
  PEST分析       →     5 Forces分析     →    3C分析
    ↓                      ↓                    ↓
 マクロトレンド把握    業界の競争環境理解    顧客・競合・自社の関係
    ↓                      ↓                    ↓
                    SWOT分析で統合整理
                          ↓
                  バリューチェーン分析
                          ↓
                   課題の特定・優先順位付け
```

---

## 1. PEST分析（マクロ環境分析）

### 目的
業界に影響を与える外部のマクロ環境要因を体系的に把握する。

### 4つの観点

| 観点 | 着眼点 | 具体的な調査項目 |
|------|--------|---------------|
| **P**olitics（政治） | 法規制・政策 | 業界関連法の改正、補助金・税制優遇、規制緩和/強化、国際条約 |
| **E**conomy（経済） | 景気・市場 | GDP成長率、為替変動、金利動向、原材料価格、消費者物価 |
| **S**ociety（社会） | 人口・価値観 | 少子高齢化、人口減少、働き方改革、SDGs意識、消費行動変化 |
| **T**echnology（技術） | 技術革新 | AI/DXの進展、特許動向、新技術の実用化、技術標準の変化 |

### 実践のコツ
- **時間軸を意識する**: 現在の状況だけでなく、3-5年後の変化を予測する
- **相互関係を見る**: 例「規制変更(P) → 新技術需要(T) → 市場拡大(E)」
- **自業界への影響度で重み付け**: 全てを均等に扱わず、影響の大きい要因を深掘り

### 検索クエリテンプレート
```
P: "{業界名} 法改正 規制 2026"  "{業界名} 政策 補助金"
E: "{業界名} 市場規模 成長率"  "{業界名} 景気影響"
S: "{業界名} 人手不足"  "{業界名} 消費者ニーズ変化"
T: "{業界名} DX AI活用"  "{業界名} 新技術 イノベーション"
```

## 2. 5 Forces分析（業界構造分析）

### 目的
マイケル・ポーターの競争戦略モデル。業界の収益性を決める5つの競争要因を分析する。

### 5つの力

```
                  新規参入の脅威
                       ↓
   売り手の    →  業界内の競争  ←   買い手の
   交渉力                           交渉力
                       ↑
                  代替品の脅威
```

| 力 | 分析のポイント | 強い場合の意味 |
|----|-------------|-------------|
| **業界内の競争** | 競合の数、差別化の程度、成長率 | 価格競争が激しく収益性低下 |
| **新規参入の脅威** | 参入障壁の高さ、必要資本、規制 | 新たな競合が出現しやすい |
| **代替品の脅威** | 代替技術の成熟度、切替コスト | 既存製品が陳腐化しやすい |
| **買い手の交渉力** | 顧客の集中度、価格感度 | 値下げ圧力が強い |
| **売り手の交渉力** | サプライヤーの集中度、切替コスト | 調達コスト上昇リスク |

### 実践のコツ
- **「力が強い = 課題がある」** と読み替える
- 各力を「高/中/低」で評価し、最も圧力の強い要因が業界の主要課題
- **AI/デジタルが各力にどう影響するか** を追加で分析すると、ITエンジニアとしての示唆が得られる

### 検索クエリテンプレート
```
競争: "{業界名} シェア 大手企業 競合"
参入: "{業界名} スタートアップ 異業種参入"
代替: "{業界名} ディスラプション AI代替"
買い手: "{業界名} 価格競争 顧客離れ"
売り手: "{業界名} サプライチェーン 原材料高騰"
```

## 3. 3C分析（ミクロ環境分析）

### 目的
顧客(Customer)・競合(Competitor)・自社/対象企業(Company)の関係性から、成功要因を特定する。

### 3つの視点

| 要素 | 分析内容 | 調査項目 |
|------|---------|---------|
| **Customer** | 顧客は誰か、何を求めているか | セグメント、ニーズ、購買行動、不満点 |
| **Competitor** | 競合はどこか、何が強いか | 主要プレイヤー、戦略、強み弱み、差別化ポイント |
| **Company** | 対象企業の立ち位置は | 経営資源、技術力、ブランド、財務状況 |

### 実践のコツ
- **KSF（Key Success Factor）** を見つけることが最終目的
- 3つの視点を個別に分析した後、**交差する領域**にビジネスチャンスがある
- 顧客起点で考え、「顧客が未充足のニーズ」× 「競合がカバーしていない領域」= チャンス

## 4. SWOT分析 + クロスSWOT

### 目的
内部環境（強み・弱み）と外部環境（機会・脅威）を統合し、戦略方向性を導く。

### SWOT マトリクス

|  | プラス要因 | マイナス要因 |
|---|----------|-----------|
| **内部環境** | Strengths（強み） | Weaknesses（弱み） |
| **外部環境** | Opportunities（機会） | Threats（脅威） |

### クロスSWOT（戦略導出）

| | 機会 (O) | 脅威 (T) |
|---|---------|---------|
| **強み (S)** | **SO戦略**: 強みを活かして機会を最大化 | **ST戦略**: 強みで脅威を回避・低減 |
| **弱み (W)** | **WO戦略**: 弱みを克服して機会を活用 | **WT戦略**: 弱みと脅威の影響を最小化 |

### 実践のコツ
- PEST → 5 Forces → 3C の結果を SWOT に統合する
- **クロスSWOT で4つの戦略方向性を出すのが最も実践的**
- ITソリューションは主に WO戦略（弱みを技術で克服）に関連することが多い

## 5. バリューチェーン分析

### 目的
業界の価値提供プロセスを分解し、各工程の課題を特定する。

### 一般的なバリューチェーン

```
[調達] → [開発/製造] → [物流] → [販売/マーケ] → [サービス/保守]
  ↑          ↑           ↑          ↑              ↑
  サプライ    品質/コスト   配送効率    顧客獲得        顧客満足
  チェーン    管理         ラストマイル チャネル         ロイヤルティ
```

### 業界別バリューチェーンの例

| 業界 | 上流 | 中流 | 下流 |
|------|------|------|------|
| 製造業 | 原材料調達 | 製造・品質管理 | 販売・アフターサービス |
| 金融業 | 商品開発 | 審査・リスク管理 | 営業・顧客管理 |
| 医療 | 研究開発 | 治験・薬事承認 | 医療提供・フォローアップ |
| 不動産 | 土地仕入れ | 建築・開発 | 販売・管理・運用 |

### 実践のコツ
- 各工程を「コスト」「時間」「品質」「人的依存度」で評価
- **IT/AIで自動化・効率化できる工程** を特定するのがエンジニア視点のゴール
- ボトルネック工程 = 最大の課題 = 最大のビジネスチャンス

## 6. As-Is / To-Be分析（Gap分析）

### 目的
現状（As-Is）と理想（To-Be）のギャップから、真の課題を特定する。

### 実践フォーマット

```markdown
| 観点 | As-Is（現状） | To-Be（理想） | Gap（課題） | 優先度 |
|------|-------------|-------------|-----------|--------|
| 業務プロセス | 紙・手作業中心 | デジタル完結 | DX推進の遅れ | 高 |
| データ活用 | 経験と勘に依存 | データドリブン | 分析基盤未整備 | 高 |
| 人材 | 属人的・高齢化 | 標準化・若手育成 | ナレッジ移転 | 中 |
| 顧客接点 | 対面・電話中心 | オムニチャネル | デジタル接点不足 | 中 |
```

### 実践のコツ
- To-Be は「業界のベストプラクティス」や「先進企業の事例」から設定する
- Gap の大きさ × 影響度 で優先順位を付ける
- **5 Whys（なぜなぜ分析）** で Gap の根本原因を深掘りする

---

## フレームワーク組み合わせパターン

### パターン1: 業界全体を俯瞰したい場合
`PEST → 5 Forces → 主要課題抽出`

### パターン2: 特定企業の機会を探したい場合
`3C → SWOT → クロスSWOT → 戦略仮説`

### パターン3: ITソリューションの機会を探したい場合
`バリューチェーン → As-Is/To-Be → ITソリューション仮説`

### パターン4: 包括的な業界深掘りをしたい場合
`PEST → 5 Forces → 3C → SWOT → バリューチェーン → As-Is/To-Be`
