## 6W2H分析 {#6w2h}

企画の全体像を漏れなく整理するための基本フレームワーク。

| 要素 | 質問 | 例 |
|------|------|-----|
| **Why** (なぜ) | なぜこの企画が必要か？ | 既存顧客の離脱率が年20%増加しているため |
| **What** (何を) | 何をするのか？ | ロイヤリティプログラムを新設する |
| **Who** (誰が) | 誰が実施するか？ | マーケティング部門＋システム部門 |
| **Whom** (誰に) | 誰のためか？ | 継続購入2回以上の既存顧客 |
| **When** (いつ) | いつ実施するか？ | 2025年Q1開始、Q3までに完全展開 |
| **Where** (どこで) | どこで実施するか？ | アプリ・ECサイト・店頭の3チャネル |
| **How** (どのように) | どのような方法で？ | ポイント付与＋ランク制度＋特別特典 |
| **How much** (いくらで) | 予算・コストは？ | 初期開発費500万円、年間運営費200万円 |

## SWOT分析 {#swot}

内部環境（強み・弱み）と外部環境（機会・脅威）を整理する。

```
            内部環境
         強み(S)   弱み(W)
外  機会(O)  SO戦略   WO戦略
部  脅威(T)  ST戦略   WT戦略
環境
```

### 各象限の活用法

| 象限 | 考え方 | 企画書での使い方 |
|------|--------|-----------------|
| SO（強み×機会） | 積極的に攻める | なぜ今チャンスかを示す根拠 |
| WO（弱み×機会） | 弱みを補強する | 課題解決の方向性を示す |
| ST（強み×脅威） | 強みで脅威を克服 | 競合優位性の根拠 |
| WT（弱み×脅威） | リスク最小化 | リスク対策セクションの素材 |

## 3C分析 {#3c}

市場環境を顧客・競合・自社の3視点から分析する。

### Customer（顧客・市場）
- ターゲット市場の規模（TAM/SAM/SOM）
- 顧客のニーズ・ペインポイント
- 購買行動・意思決定プロセス
- 市場トレンド・成長率

### Competitor（競合）
- 主要競合の特定
- 競合の強み・弱み・戦略
- 自社との差別化ポイント
- 参入障壁の分析

### Company（自社）
- 自社のコアコンピタンス
- 活用できるリソース・アセット
- ブランド・信頼性
- 実行可能性の根拠

## 業種別構成パターン {#業種別}

### 新規事業企画書
```
1. エグゼクティブサマリー
2. 市場機会・課題
3. ビジョン・ミッション
4. 事業コンセプト（ビジネスモデル）
5. ターゲット市場と顧客セグメント
6. 競合分析・差別化戦略
7. 収益モデル・財務計画
8. ロードマップ・フェーズ計画
9. チーム・実行体制
10. リスクと対策
11. 資金調達計画（必要な場合）
```

### イベント企画書
```
1. イベント概要（タイトル・コンセプト）
2. 開催目的・KPI
3. ターゲット参加者
4. 開催概要（日時・場所・形式・規模）
5. コンテンツ・プログラム
6. 集客計画（告知方法・集客目標の積み上げ）
7. 運営体制（担当者・役割分担）
8. 予算内訳
9. スケジュール（準備〜事後対応）
10. リスク対策
```

### 商品・サービス開発企画書
```
1. エグゼクティブサマリー
2. 市場背景・顧客課題
3. 商品・サービスコンセプト
4. ターゲット顧客
5. 機能・特徴・差別化点
6. 市場検証データ（ユーザーインタビュー・調査結果）
7. 開発ロードマップ
8. 価格戦略・収益計画
9. マーケティング計画
10. リソース・予算
```

### マーケティング・販促企画書
```
1. 現状分析と課題
2. 施策目的・KPI
3. ターゲット（ペルソナ）
4. 施策コンセプト・訴求メッセージ
5. 具体的施策内容（チャネル・クリエイティブ）
6. 実施スケジュール
7. 予算配分（チャネル別ROI）
8. 効果測定方法
```

### 業務改善提案書
```
1. 現状の課題と損失（定量化）
2. 改善の目的・目標
3. 改善案の内容
4. 実施ステップ
5. 期待効果（コスト削減・時間短縮）
6. 必要なリソース・コスト
7. スケジュール
8. リスクと対策
```

## SMARTゴール設定 {#smart}

| 要素 | 意味 | チェック質問 |
|------|------|-------------|
| **S**pecific（具体的） | 何を達成するか明確か | 「売上向上」ではなく「月次売上20%増加」 |
| **M**easurable（測定可能） | 数値で測れるか | KPIが定義されているか |
| **A**chievable（達成可能） | 現実的に届くか | リソースと照らして実現性があるか |
| **R**elevant（関連性） | 事業目標と整合しているか | なぜこの指標が重要かを説明できるか |
| **T**ime-bound（期限付き） | いつまでに達成するか | 明確な期日があるか |

## リスクマトリクス {#リスク}

各リスクを「発生確率」×「影響度」で評価し、対策の優先度を決める。

```
        影響度
        低    中    高
発  高  中    高    最高
生  中  低    中    高
確  低  最低  低    中
率
```

### リスク記載テンプレート

| リスク | 発生確率 | 影響度 | 優先度 | 対策 | 担当 |
|--------|----------|--------|--------|------|------|
| （例）競合の先行参入 | 中 | 高 | 高 | 開発スピードを上げる / 独自機能の特許申請 | 開発部門 |
| （例）予算超過 | 低 | 中 | 中 | 予備費10%確保 / フェーズ分けで変動費を管理 | 財務部門 |

## 企画書の説得力を高める追加テクニック

### ストーリーフレームワーク（PREP法）

説得力のある段落・スライドを書くときの型：

```
P = Point（結論・主張）
R = Reason（理由）
E = Example（具体例・データ）
P = Point（結論の再提示）
```

### コントラスト効果の活用

「現状 vs 企画実施後」を並べてビフォー/アフターを可視化する。読み手は差分で価値を判断する。

### 損失回避バイアスの活用

人間は「得る利益」より「失う損失」に2.5倍敏感（行動経済学）。
「この企画で○○が得られる」だけでなく、「この企画をやらないと○○を失う」という視点も必ず入れる。
