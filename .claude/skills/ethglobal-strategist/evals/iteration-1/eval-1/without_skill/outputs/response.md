## 2. 差別化ポイントの分析

### 弱い差別化（避けるべきアプローチ）

- 「自然言語でスワップできる」だけでは不十分（既存プロダクトで実現済み）
- 「複数DEXを比較してベストレートで実行」だけでは差別化にならない
- UIの見た目を工夫するだけでは評価されにくい

### 強い差別化ポイント（推奨）

#### A. **リスク管理・ガードレールAI**
- 操作前に「スリッページリスク」「インパーマネントロスの試算」「ガス代最適タイミング」を自然言語で警告
- 「今このプールに流動性追加するのは〇〇のリスクがある」と事前説明するエージェント
- 既存プロジェクトの多くは「実行」に特化しており、**リスク説明レイヤー**が弱い

#### B. **マルチステップ戦略の自動実行（Agentic Automation）**
- 「ETHを担保にDAIを借りて、そのDAIをyearnに入れて最適利回りを得て、利息が出たら自動でETHを買い戻す」といったマルチステップ戦略を1文で設定
- 現状の類似プロジェクトは**1回のトランザクション完結型**が多く、**継続的・条件分岐型**のAgentはまだ希少

#### C. **ソーシャル/コンテキスト認識**
- ウォレットの過去の取引履歴を分析し、「このユーザーはリスク回避型」「高頻度トレーダー」などのプロファイリングを行い、それに合わせた提案をする
- DeFiの「パーソナライゼーション」は未開拓領域

#### D. **クロスチェーン×自然言語**
- 「Baseに持っているUSDCをArbitrumのAaveに預けて」という指示を、ブリッジ→デポジットの一連として自動実行
- LI.FI、Axelar、CCTP等のブリッジプロトコルと統合したクロスチェーンAgentは差別化余地がある

#### E. **説明可能なDeFi（Explainable DeFi）**
- 実行後に「なぜこのルートを選んだか」「ガスをいくら節約したか」をナチュラルランゲージで報告
- 監査トレイルとして活用できる「AIの意思決定ログ」

## 3. アイデアブラッシュアップの具体的提案

### 推奨テーマ：「Intent-Aware DeFi Guardian Agent」

#### コアコンセプト
> ユーザーの意図を理解し、リスクを説明し、マルチステップ戦略を安全に自動実行するDeFiコパイロット

#### 主要機能（MVP for Hackathon）

1. **自然言語インテント解析**
   - "swap 1 ETH to USDC" → 実行計画生成
   - "provide liquidity to ETH/USDC pool" → リスク試算付きで提案

2. **リアルタイムリスクレポート**（差別化の核心）
   - 実行前に「現在のスリッページは0.3%」「IL試算: ETH±20%で約2%の損失」を日本語/英語で説明
   - ユーザーが「OK」と言ったら実行

3. **条件トリガー型Automation**
   - 「ETHが$3000を下回ったらDAIに換えて」という条件付き実行の設定
   - Keeper Network / Gelato / Chainlink Automation との統合

4. **実行ログのNL要約**
   - 実行後「昨日比でガスを15%節約しました。最適ルート: Uniswap v3 → 1inch」などを報告

#### 技術スタック（推奨）

| レイヤー | 技術選択 |
|---|---|
| LLM | Claude 3.5 Sonnet / GPT-4o (Tool calling活用) |
| Agent Framework | LangChain / Coinbase AgentKit |
| DEX Integration | Uniswap v3/v4 SDK, 1inch API |
| Cross-chain | LI.FI SDK / CCTP |
| Automation | Gelato Network / Chainlink Automation |
| Wallet | EIP-4337 (Account Abstraction) + Safe |
| Oracle | Chainlink Price Feeds |
| Frontend | Next.js + wagmi |

#### スポンサー賞の狙い目

| スポンサー | 賞 | 戦略 |
|---|---|---|
| **Uniswap Foundation** | Uniswap v4 Hooks活用 | v4 HooksでAgent専用のカスタムロジックを実装 |
| **Coinbase** | AgentKit活用 | Coinbase AgentKitを中核に据える |
| **Chainlink** | Automation / CCIP | 条件トリガー実行にChainlink Automationを使用 |
| **1inch** | Fusion+ Integration | 1inchのインテントベース注文をAgentで活用 |
| **LI.FI** | Cross-chain routing | クロスチェーン戦略にLI.FI統合 |

### スコアの解説

**現状の問題点（4.6点）：**
- 「自然言語でDeFi操作」はすでに複数プロダクトが実現済みであり、ETH GlobalのJudgeには「見たことある」と思われるリスクが高い

- リスク説明AI + 条件実行 + マルチチェーンの組み合わせは新規性が高い
- Uniswap v4 Hooks、Coinbase AgentKit、Gelato等複数スポンサー賞を同時に狙える
- デモが映える（「この指示を入れると、こんな詳しいリスク解析が出て、自動実行される」）

## 6. ピッチの推奨ストーリーライン

1. **Problem**: DeFiは複雑すぎて一般ユーザーには難しい。しかし既存のAI DeFiツールは「実行」するだけで、**リスクを教えてくれない**。
2. **Solution**: 自然言語で指示するだけで、リスクを説明し、承認後に安全に実行するDeFiコパイロット。
3. **Demo**: ライブデモで「ETHをUSDCにスワップして」→ リスク説明表示 → ユーザー承認 → 実行完了
4. **Tech**: LLM + AgentKit + Uniswap v4 + Gelato
5. **Traction**: 「DeFi利用者の〇〇%がリスク理解なしに操作している」等のデータ引用
