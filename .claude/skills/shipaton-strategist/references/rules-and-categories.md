## 参加要件（失格を避けるための必須事項）

1. **新規アプリであること**: iOS / iPadOS / macOS / Android 向け。初回公開が提出期間内（8/1〜9/30）であること。**既存アプリのアップデートは対象外**
2. **RevenueCat SDK の統合**: 少なくとも1つのアプリ内課金（in-app purchase / web purchase）を RevenueCat 経由で実装、または RevenueCat Ads で広告配信すること
3. **ストア公開**: App Store / Google Play / Samsung Galaxy Store のいずれかに公開済みであること
4. 参加者は居住国の成年年齢以上であること（一部の国・地域は除外）

## 賞カテゴリ（9つ）と狙い方

### 1. Grand Prize（1位 $100,000 / 2位 $20,000 / 3位 $10,000）
- **審査基準**: ユーザートラクションとグロースの勢い。提出期間中の実績（DL数、課金ユーザー、売上、レビュー）が最重要
- **狙い方**: プロダクトの質だけでは獲れない。8月上旬に早期リリースして9月をまるごとグロース期間に充てる。市場の広い課題×低CACのバイラル要素が必要。難易度は最高

### 2〜8. カテゴリ賞（各賞 1位 $15,000 / 2位 $10,000 / 3位 $5,000）

| 賞 | 審査基準 | 狙い方 |
|---|---|---|
| **HAMM Award** | 最も賢く堅牢な RevenueCat 収益化実装 | 課金モデル自体に思想を持たせる。単純なサブスクではなく、価格設計・ペイウォールUX・Entitlements 活用に「語れるストーリー」を作る（例: 2025年の Vector Guard「1:50 ジャスティスモデル」= 課金1件で高リスク者50人に無償提供） |
| **RevenueCat Design Award** | 卓越したデザイン・アニメーション・クラフト | 機能を絞り、マイクロインタラクションと動きの質に時間を投資。「おっ」と思わせる1画面を作り込む。デザイナー不在ならこの賞の主軸は避ける |
| **RevenueCat Peace Prize** | 最大の社会的インパクト | 健康・教育・アクセシビリティ・安全。当事者性のあるストーリーが強い。マネタイズとミッションの整合（HAMM との相乗り）が高評価 |
| **Best Game Award** | 最高のモバイルゲーム | ゲームは開発コストが高くレッドオーシャン。ゲーム開発経験者以外には非推奨 |
| **#BuildInPublic Award** | 開発過程の公開ストーリーテリング | X/LinkedIn/TikTok での継続発信。数値・失敗・学びを透明に共有し、フィードバックを実際に反映した証跡を残す。**どのアイディアとも相乗り可能で、グロースにも直結するため全員推奨** |
| **Catvertising Award** | RevenueCat Ads の最も創造的な活用 | 2026年の新機軸（広告マネタイズ）。競合が少ない可能性が高く、広告×課金のハイブリッド設計で狙い目 |
| **Next Gen Award** | 学生限定 | ストア公開・開発者アカウント不要（動画＋ソースコード提出）。学生なら参加障壁が最も低い |

### 9. Conflict of Interest Award
RevenueCat・スポンサー社員向け。一般参加者は対象外。

## 一般的な審査観点（全カテゴリ共通の土台）

- **Innovation**: 市場に対する新規性（新しいアイディア・機能・ユースケース）
- **Execution**: 実装の完成度・安定性・視覚的一貫性
- **Feasibility**: ハッカソン後も現実的に成長・継続できるか
- **Integration**: RevenueCat SDK / API の効果的な活用

## カテゴリ選定の指針

1. **主軸1つ＋副次1〜2つ**を選ぶ。全方位狙いは全敗する
2. 主軸はユーザーの強み（デザイン力・マネタイズ思想・当事者性・発信力）と一致させる
3. #BuildInPublic は常に副次として乗せる
4. グランドプライズは「結果として獲れたら嬉しい」位置付けにし、設計段階ではカテゴリ賞に最適化する方が期待値が高い（競合812作品→608提出の中で戦うため）
