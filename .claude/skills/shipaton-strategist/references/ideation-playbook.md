# アイディア出し・検証プレイブック

情報源: RevenueCat 公式「How to win Shipaton, part 1: coming up with an idea」（70回以上のハッカソン出場・45勝の著者による）

## 勝てるアイディアの3要素

1. **本物の課題**を解決している（想像上の課題ではない）
2. **実在する人々**のための課題である
3. 使った瞬間に**「aha」体験・喜び**がある

到達すべき心境: 「これは自分が作らなければならない。正しく作れるのは自分だけだ」

## 課題の見つけ方

### やること
- **自分がよく知るドメインから始める**: 職業、趣味、家族の状況、住んでいる地域。語れる当事者性があるほどピッチも強くなる
- **実際のユーザーと話す**: オープンクエスチョンと仮定の質問を使う。自分のアイディアを売り込むのではなく、純粋な好奇心で聞く
- **1つの課題に集中する**: 理髪店予約アプリに請求管理と分析を足して2ヶ月の計画が6ヶ月になり、その間に競合が出現した実例がある。Shipaton の2ヶ月では致命傷になる

### 避けること（敗因ワースト）
- 自分の課題を「みんなの課題」と思い込む（Twitch賭けアプリの実例: ソリューション先行で作った後、視聴者は受動的に見たいだけと判明）
- 「作りたいもの」を作る（「作る必要があるもの」ではなく）
- 複数の課題を同時に解こうとする
- 検証前の過剰設計
- ユーザーフィードバックの無視、1人のユーザーの特殊事情への過剰適応

## 発散フェーズの型

候補は5〜7案。各案を必ずこの1行（ロジックライン）に落とす:

> 「**[誰]** のための **[アプリ名]** は、**[独自のアプローチ]** によって **[片付けたいジョブ]** を解決し、**[価値ある結果]** をもたらす」

この1行が書けないアイディアは、デモ動画の最初の15秒でも説明できない = 審査で負ける。

発散の起点リスト:
- ユーザーの職業・専門知識で「外部の人は知らない不便」は何か
- 家族・友人が繰り返し愚痴っていることは何か
- お金・健康・時間の「見えない損失」を可視化できないか（Payout 型）
- 既存の面倒な制度・手続きを自動化できないか
- 話題にしたくなる・スクショを共有したくなる切り口はあるか（ReadHim 型）
- 課金することが誰かへの貢献になる設計は可能か（Vector Guard 型）

## 収束フェーズ: スコアリング表

| 評価軸 | 1点 | 5点 |
|---|---|---|
| 課題の切実さ | あれば便利 | お金を払ってでも今すぐ解決したい |
| 2ヶ月MLP実現性 | バックエンド＋ML基盤が必要 | 1機能・ローカル完結で作れる |
| 賞カテゴリ適合 | どの賞の基準にも中途半端 | 主軸カテゴリの審査基準に直撃 |
| マネタイズの自然さ | 課金理由を後付けする必要がある | 課金が体験の一部として自然 |
| 15秒説明可能性 | デモに文脈説明が必要 | 画面を見せた瞬間に伝わる |
| 本人の熱量 | 流行っているから | 当事者として2ヶ月語り続けられる |

合計点で機械的に決めず、「実現性」と「熱量」に低スコアがある案は合計が高くても外す（この2つは後から上げられない）。

## Minimum Lovable Product（MLP）

MVP ではなく MLP を作る。「動く最小限」ではなく「愛される最小限」。

- **スコープはケチり、磨きは惜しまない**: プロダクトの価値を示す最小の表面積を作り、そこを徹底的に磨く
- 自問リスト:
  - 「この課題を解決する最小のものは何か？」
  - 「これを個人的・独自に感じさせるものは何か？」
  - 「どこを磨けば『わ、これは予想外』と言わせられるか？」
- 目指すのは「城ではなく、味のある小屋」

## 継続的検証

検証は一度きりではなく2ヶ月間続くループ。

1. **プロトタイプ段階**: Figma 等を見せ、ユーザーに声に出しながら操作してもらう
2. **初期ビルド段階**: 自分のスマホを渡して直接触ってもらう
3. **配布可能段階**: TestFlight 等でインストールしてもらい、後日フォローアップ通話

- 「少し恥ずかしい」と感じる段階でリリースする（恥ずかしさは、無駄な機能に資源を使っていない証拠）
- この段階では分析ツールより定性フィードバック。量より質
- **主要メトリクスを1つ決める**: Activation（すぐコア価値に到達したか）/ Retention（戻ってきたか）/ Referral（人に薦めたか）/ Monetization（払う気があるか）
- 検証が「課題が本物でない」「誰も払わない」を示したら、**捨てて最初からやり直す勇気を持つ**。8月前半なら十分間に合う
