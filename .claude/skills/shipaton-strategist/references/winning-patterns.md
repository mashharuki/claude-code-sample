# Shipaton 2025 勝者分析 — 勝ちパターン集

情報源: RevenueCat 公式ブログ「Shipaton 2025 Winners」、note「Shipaton 2025受賞作品から学ぶ」（AI-Driven Lab）
2025年実績: 参加数万人、提出812プロジェクト（ショーケース掲載608アプリ）。

## 2025年 主要受賞作品

### グランプリ（Build & Grow Award）: Payout
集団訴訟の和解金（class-action settlement）を探して申請できるアプリ。不透明な法的プロセスを透明化・自動化。
- **全コードを Claude Code と Cursor で生成。手書きコードゼロ**
- 審査で評価された点: 新規性、実装品質、実現可能性、SDK統合の巧みさ
- 教訓: 「誰もが知らないが切実な金銭的メリット」を掘り当てた課題選定の勝利

### HAMM Award（マネタイズ）: Vector Guard
感染症を媒介する虫（disease vector）の識別アプリ。
- **「1:50 ジャスティスモデル」**: $2.99 のプレミアム課金1件ごとに、高リスク地域のユーザー50人へ無償アクセスを自動提供
- 教訓: 課金モデル自体がプロダクトのストーリーになっている。HAMM は「いくら稼いだか」ではなく「課金設計の思想」で獲れる
- 2位 Napkinmatic AI3D、3位 Kigaru Talks（日本語会話練習アプリ — 日本発でも獲れる証拠）

### Peace Prize（社会的インパクト）: Heartbeat Hero
CPR（心肺蘇生）トレーニングアプリ。
- 音声ガイド・ハプティクス・フラッシュライト対応など、ADHD・ディスレクシアのユーザーも考慮した**アクセシビリティファースト設計**
- 2位 Hearing Buddy（リアルタイム字幕）、3位 MoodHaven（子どものメンタルヘルス記録）
- 教訓: 健康・教育・アクセシビリティ領域×当事者性が強い

### Design Award: Dayloop（毎日のタイムラプス）
- 技術的な複雑さを直感的なインターフェースの裏に隠す設計が評価された
- 2位 SkillMe、3位 PitchLab（野球の投球トラッキング）

### #BuildInPublic Award: Gurwi – Learn Anything
コロンビア発の教育プラットフォーム。
- 開発過程の動画が累計20万回再生。コミュニティのフィードバックを実際に反映
- ストア評価 4.9/5（1,000件以上のレビュー）
- 2位 Echo Reminder（音声→タスク変換）、3位 Tomo Japan（日本旅行ガイド）
- 教訓: 「作っている過程」自体がマーケティングでありプロダクト改善ループになる

### Buzziest Launch: ReadHim（AIテキストメッセージ分析）
- 2位 Shutter Declutter（写真整理）、3位 MemoLune（音声ファーストのメモ）
- 教訓: SNSでバズる「話題にしたくなる切り口」（恋愛×AI分析など）はローンチの爆発力を生む

### Kotlin Multiplatform Award: Momental（瞑想アプリ）
- 小規模チームが KMP で iOS/Android 両対応を効率的に実現

## 横断的な勝ちパターン（2026年戦略に翻訳）

1. **AI駆動開発が標準**: 優勝者が全コードAI生成。人間の価値は「何を作るか」「品質判断」「ユーザー理解」に移った。→ 開発速度で差はつきにくい。**課題選定とストーリーで差がつく**
2. **社会的価値×収益の両立**: 受賞作の多くが健康・教育・アクセシビリティ。「儲かるが意義もある」はもはや標準装備
3. **ミッション連動型マネタイズ**: 1:50モデルのように、課金するとミッションが前進する設計。HAMM と Peace Prize の同時狙いが可能
4. **Build in Public は勝者の共通行動**: 賞そのものに加え、グロース（= グランプリ基準）とレビュー獲得に直結
5. **本物のストーリーテリング > 洗練されたマーケ**: 個人の当事者性・失敗談を含む発信が勝つ
6. **1機能に絞った小さなアプリが多い**: 音声メモ、写真整理、タイムラプス…。「小さく作って深く磨く」が受賞ライン
7. **日本発でも勝てる**: Kigaru Talks（HAMM 3位）、Tomo Japan（BuildInPublic 3位）
8. **収益の実例**: 過去優勝者は複数アプリで累計$1M超。Shipaton用に約2週間で作ったアプリが数ヶ月で月$45,000に成長した例もある

## アイディア出しの際にこのリストを使う方法

- 受賞作と同じアプリを作らない（既に存在する = 新規性なし）
- 代わりに「パターン」を抽出して転用する: 「不透明な制度×自動化」（Payout型）、「当事者性のあるヘルスケア×アクセシビリティ」（Heartbeat Hero型）、「話題にしたくなるAI分析」（ReadHim型）、「課金=社会貢献」（Vector Guard型）
- ユーザー自身の経験・職業・生活から、これらのパターンに当てはまる未開拓の課題を探す
